Steps on how to create and manage a GitHub repository. Git terminologies are highlighted.

  1. Create a GitHub account - Visit github.com
  2. Sign in to github.com - Visit github.com/login
  3. Create a Repository (Repo) - A project folder. Use lowercase letters and hyphen to separate compound words (Eg. repo-name).
    • Rename repo - Select repo > Settings tab > "Repository name" label > enter new name in field provided > click [Rename] to continue.
    • Delete repo - Select repo > Settings tab > "Danger Zone" fieldset (below) > click [Delete this repository] > follow the prompted instructions to continue.
  4. Download and install GitHub desktop software - Visit desktop.github.com
  5. Clone a Repository - Copy a repository (Master) created on the GitHub website to a repository (Local) on your computer to continue
    • via GitHub desktop - Sign in to github desktop app, confirm display name and email, select repo from list of repositories, click [Clone this repository], select any workspace folder on computer
    • via GitHub website - Sign in to github website, select repo, click [Clone or download] green button > select Open in Desktop
  6. Work on project - Create new files in your local repo or modify existing repo files where applicable; using any text-editor or IDE of your choice (preferably, VS Code).
    • Commit changes - Open github desktop app, select repo, foreach modified file(s) (see left margin), enter "Change Title" (bottom-left margin), enter "Change Description" (bottom-left margin), click [Commit to master], select [Push origin] to update remote repository
    • Review changes - Sign in to github website, select repo, repo file changes are automatically listed in the default tab (i.e. Code tab)
  7. Add Collaborator(s) - The software team (being that this is a private repository). Sign in to github website > select repo > Settings tab > Manage access menu > click [Invite a collaborator] green button > follow the prompted instructions to continue
